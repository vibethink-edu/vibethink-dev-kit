# _vibethink_doc (Knowledge Repository)

> [!IMPORTANT]
> This repository is the **Central Brain** of VibeThink. It consolidates Methodology, Knowledge Base, and External Standards.

## 🌍 Philosophy
This repository contains the **Universal Standards** for VibeThink. It is decoupled from any specific implementation (like the Dev-Kit) to ensure it can govern multiple projects and technologies.

## 📂 Structure

### 🧠 vthink-methodology
*The core VibeThink Standard.*
- **01_PRINCIPLES**: Core values and team organization.
- **03_PROCESSES**: Workflows and methodologies.
- **05_BEST_PRACTICES**: Quality standards.
- **VTHINK_*.md**: Core definitions.

### 📚 knowledge-base
*General documentation and guides.*
- **guides**: Practical guides for developers.
- **development**: Development notes and logs.
- **architecture**: High-level architectural decisions.
- **evaluations**: Technology assessments.

### 🌐 external-standards
*Adopted industry standards.*
- **CMMI**: CMMI Level 3 Guidelines.
- **OpenSpec**: (Placeholder) OpenSpec specifications.
- **Speckit**: (Placeholder) Speckit documentation.

## 🚀 How to Use
1.  **Move this folder** out of `vibethink-orchestrator-main` to your workspace root.
2.  **Initialize it** as a new git repository if desired.
3.  **Reference it** in your other projects (Dev-Kit, Orchestrator) as the "Source of Truth".
